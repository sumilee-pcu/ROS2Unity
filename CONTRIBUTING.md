# 기여 지침

## 작업 원칙

- 교재 설명과 실행 코드는 같은 변경에서 함께 갱신합니다.
- 명령은 ROS 2 Jazzy와 Ubuntu 24.04를 기준으로 검증합니다.
- 검증하지 않은 기능은 문서에 `계획`, `실험`, `검증 완료` 중 하나로 표시합니다.
- 외부 교재·블로그의 문장을 복사하지 않고 직접 설명합니다.
- Unity Asset Store 자산과 라이선스가 불분명한 모델은 커밋하지 않습니다.

## 브랜치

- 기능: `feature/<name>`
- 교재: `docs/<chapter>`
- 수정: `fix/<name>`

## 완료 기준

1. 관련 명령 또는 빌드가 실행된다.
2. 예상 토픽·노드·로그가 문서와 일치한다.
3. 실패했을 때의 점검 방법이 교재에 포함된다.
4. 출처와 버전을 기록한다.
