# 첫 토픽 왕복 실습

## 1. 목표

- Jazzy에서 발행한 heartbeat를 Unity가 구독한다.
- Jazzy 또는 Unity에서 발행한 `/cmd_vel`을 상대편이 수신한다.
- 연결 성공과 로봇 이동 성공을 분리하여 확인한다.

## 2. ROS 측 단독 확인

```bash
docker compose up --build
```

다른 터미널:

```bash
docker compose exec ros2unity bash
ros2 topic echo /ros2unity/heartbeat
```

예상 출력:

```yaml
data: jazzy-ok:0
---
data: jazzy-ok:1
```

## 3. `/cmd_vel` 수신 확인

```bash
ros2 topic pub --once /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.2}, angular: {z: 0.5}}"
```

컨테이너 로그에 다음 값이 나타나면 ROS 측 준비는 완료다.

```text
cmd_vel received: linear.x=0.200 angular.z=0.500
```

## 4. Unity 물체 제어

Unity 씬의 로봇 오브젝트에 다음 컴포넌트를 추가한다.

- `Rigidbody`
- `DifferentialDriveController`
- `RosSharpTwistSubscriber`

`RosSharpTwistSubscriber`의 토픽을 `/cmd_vel`로 설정한다. Unity Play 후 앞의 `ros2 topic pub` 명령을 실행한다.

## 5. 성공 기준

- 선속도만 주면 Unity 로봇이 전진한다.
- 각속도만 주면 y축을 중심으로 회전한다.
- 명령을 중단하면 0.5초 안에 멈춘다.
- 컨테이너가 재시작되어도 같은 절차로 재현된다.

## 6. 아직 실제 검증이 필요한 부분

저장소 초기 버전의 C# subscriber는 ROS# 2.2.2 API를 기준으로 작성했다. Unity Editor 컴파일과 실제 WebSocket 수신은 M1 실험에서 검증하고, API 차이가 발견되면 코드와 본 장을 함께 수정한다.
