# 개발환경 구축

## 1. 준비물

- Git
- Docker Desktop 또는 동등한 컨테이너 실행 환경
- Unity Hub
- Unity 2022.3.62f3

첫 검증에서는 Unity 6보다 ROS#가 개발 기준으로 삼은 Unity 2022.3 LTS를 우선한다. 이 저장소는 Personal 라이선스로 사용할 수 있고 2022 LTS 보안 수정이 포함된 `2022.3.62f3`을 기준 버전으로 고정한다. macOS Apple Silicon에서 검증했으며 이동용 Windows 노트북에서도 같은 에디터 버전을 사용한다.

## 2. Jazzy rosbridge 실행

저장소 루트에서 실행한다.

```bash
docker compose up -d --build
```

정상 로그의 핵심은 다음 두 문장이다.

```text
Rosbridge WebSocket server started on port 9090
Bridge probe ready
```

컨테이너 안에서 노드를 확인한다.

```bash
docker compose exec ros2unity bash
ros2 node list
ros2 topic list
```

예상 항목:

```text
/bridge_probe
/rosbridge_websocket
/cmd_vel
/ros2unity/heartbeat
```

## 3. Unity 프로젝트 열기

Unity Hub에서 저장소의 다음 폴더를 프로젝트로 연다.

```text
unity_ros_bridge_2022
```

최초 실행 시 Package Manager가 `Packages/manifest.json`에 고정된 ROS#를 자동 설치한다. 데모 씬은 다음 경로에 있다.

```text
Assets/Scenes/ROS2UnityBridge.unity
```

씬을 열고 Play를 누른다. Game 화면 왼쪽 위에 `rosbridge: CONNECTED`와 `jazzy-ok:` heartbeat가 표시되면 연결 성공이다.

## 4. ROS# 설치 정보

이 프로젝트에는 다음 Git URL이 이미 등록되어 있다. 새 프로젝트를 직접 만들 때만 Unity Package Manager의 `Add package from git URL`로 추가한다.

```text
https://github.com/siemens/ros-sharp.git?path=/com.siemens.ros-sharp#2.2.2
```

버전을 고정하지 않으면 교재 작성 이후 API나 샘플 구성이 바뀔 수 있다.

`Project Settings > Player > Scripting Define Symbols`의 Standalone 항목에는 `ROS2`가 설정되어 있어야 한다.

## 5. WebSocket 주소

Unity와 Docker가 같은 호스트에 있으면 다음 주소를 사용한다.

```text
ws://localhost:9090
```

Unity가 다른 컴퓨터에 있으면 `localhost`가 아니라 Jazzy 호스트의 LAN IP를 사용하고, 방화벽에서 TCP 9090 접근을 허용해야 한다. rosbridge 포트를 인터넷에 직접 노출하지 않는다.

## 6. 데모 씬을 다시 만드는 방법

씬을 실수로 변경했거나 삭제했다면 Unity 메뉴에서 다음을 실행한다.

```text
ROS2Unity > Build Demo Scene
```

## 7. 연결 문제 점검 순서

1. `docker compose ps`에서 컨테이너가 실행 중인지 확인한다.
2. `ros2 topic echo /ros2unity/heartbeat`가 출력되는지 확인한다.
3. 호스트에서 9090 포트가 열렸는지 확인한다.
4. Unity `RosConnector` 주소와 프로토콜을 확인한다.
5. ROS#가 ROS 2 메시지 모드로 설정되었는지 확인한다.
6. Unity Console의 첫 예외부터 해결한다.
