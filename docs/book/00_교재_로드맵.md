# 프로젝트 방향과 교재 로드맵

## 1. 목표

이 교재의 목표는 Unity 사용법만 설명하는 것이 아니다. ROS 2 Jazzy에서 이미 배운 토픽, 좌표계, 센서, SLAM, Nav2를 Unity 시뮬레이션에 다시 연결하여 시뮬레이터가 바뀌어도 유지되는 로봇 소프트웨어 구조를 이해하는 것이다.

## 2. 선수 학습

- ROS 2 노드·토픽·서비스·액션
- `/cmd_vel`, `/odom`, `/tf`, `/scan`
- URDF와 로봇 좌표계
- Gazebo Harmonic과 RViz2의 역할 차이
- SLAM Toolbox와 Nav2의 기본 데이터 흐름

## 3. 학습 단계

### 1부: 연결

1. Unity와 ROS 2의 역할
2. Jazzy rosbridge 컨테이너
3. ROS# 설치와 WebSocket 연결
4. 문자열 토픽 왕복
5. `/cmd_vel`로 Unity 물체 제어

### 2부: 이동 로봇

6. Unity 좌표계와 ROS REP-103 변환
7. 차동구동 로봇
8. 오도메트리와 `/tf`
9. URDF 모델 가져오기
10. 물리 파라미터와 시간 동기화

### 3부: 센서와 자율주행

11. 2D LiDAR와 `/scan`
12. 카메라와 이미지 메시지
13. IMU와 노이즈
14. RViz2 시각화
15. SLAM Toolbox
16. Nav2 목표 주행

### 4부: Physical AI

17. ML-Agents 기초
18. 보상 설계와 병렬 환경
19. ROS 정책과 학습 정책의 연결
20. 도메인 랜덤화와 Sim-to-Real

### 부록

- Unity 공식 ROS-TCP 포팅 실험
- `ros2-for-unity` Jazzy 소스 빌드 실험
- rosbridge와 네이티브 DDS 성능 비교
- macOS·Windows·Linux 네트워크 문제 해결

## 4. 단계별 성공 기준

각 실습은 화면이 보이는 것만으로 완료하지 않는다.

1. 노드와 토픽이 예상대로 조회된다.
2. 메시지 필드와 단위가 설명과 일치한다.
3. Unity와 ROS의 좌표 변환을 수치로 확인한다.
4. 재시작 후에도 같은 절차로 재현된다.
5. 실패 원인을 로그와 CLI로 구분할 수 있다.
